# Development Log

## Milestone M4.1 — Scheduling Engine

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-07

---

# 1. Objective

Implement the scheduling pipeline end to end with a real OR-Tools CP-SAT solver,
including the resource-assignment dimension (operations placed on capable
resources).

Implements Phase 4 of the Bootstrap Plan, the Planning Model / Scheduling Model
docs, the Constraint Framework, and ADR-005 (Scheduling Model as Anti-Corruption
Layer), ADR-006/007 (Constraint vs Objective Model).

---

# 2. Scope

Pipeline: PlanningProblem -> validate -> SchedulingModel -> SolverAdapter ->
SchedulingSolution -> Assignments.

- PlanningProblem (immutable, framework-free): resources, operations, policies.
- SchedulingModel: Resource Graph (`SchedulingResource`), Operation Graph
  (`Task` with duration, precedence, `required_capability`).
- SolverAdapter interface; OR-Tools CP-SAT implementation (only module importing
  ortools).
- Constraint categories implemented:
  - Dependency (finish-to-start, lag 0),
  - Capability (assignment only to eligible resources),
  - Resource (no-overlap per resource; one task at a time),
  - Cardinality (each task assigned to exactly one resource).
- Objective: minimize makespan (interim; see Completion Record).
- AssignmentBuilder rebuilds Assignments (operation -> resource + start/end).
- SchedulingEngine facade (stateless, constructor injection).

# 4. Out of Scope

- Constraint categories Qualification, Calendar, Policy.
- Objective Model goals requiring unmodelled entities (demand completion,
  utilization, workload balance).
- Building the problem from persisted instances / Planning Context.

---

# 5. Acceptance Criteria

- Generate Schedule produces valid Assignments for a feasible problem.
- An operation is only assigned to a resource whose capability set matches.
- A resource performs one operation at a time.
- Precedence is respected.
- Infeasible problems (no capable resource) are reported, not crashed.

---

# 6. Verification

- Engine pipeline via FakeSolverAdapter: assignments, empty/unknown-dependency
  rejection, capability-aware assignment, infeasible.
- Real CP-SAT (`solver/tests`): precedence + optimal makespan; capability
  matching; single-resource serialization (makespan 7); two-resource
  parallelism (makespan 4); no-capable-resource infeasible.
- End-to-end API: operation requiring a capability assigned to the only capable
  resource.

---

# 8. Completion Record

**Date:** 2026-07-07

The solver boundary is preserved: only `ortools_solver_adapter.py` imports
OR-Tools; the SchedulingModel and SchedulingSolution are framework-free.

**Interim objective (ADR-007):** the solver minimizes makespan. The Objective
Model doc lists demand-completion / utilization / workload-balance objectives,
which require entities not yet modelled. Makespan is the only sensible objective
at this stage and is isolated in the adapter, to be revisited when Demand and the
Objective Profile land.
