# Development Log

## Milestone M13.1 — PyWebView Desktop Packaging

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-08

---

# 1. Objective

Package the app as a desktop application (ADR-011): Flask serves the built SPA
and a PyWebView window hosts it, so the same backend runs on desktop or web.

---

# 2. Scope

- `register_frontend` in the app factory serves `frontend/dist` when present,
  with SPA fallback to index.html; API paths never resolve to the SPA (JSON 404
  preserved). No-op when the build is absent (dev uses Vite).
- `desktop.py`: runs Flask in a background thread and opens a PyWebView window;
  degrades to a plain server + instructions if pywebview is not installed.
- `pywebview` added to requirements.

---

# 3. Deliverables

| Component     | Location                                 |
| ------------- | ---------------------------------------- |
| SPA serving   | `backend/app.py` (register_frontend)     |
| Desktop entry | `desktop.py`                             |
| Dependency    | `requirements.txt` (pywebview)           |
| Tests         | `backend/tests/test_frontend_serving.py` |

---

# 4. Out of Scope

- A frozen single-file installer (PyInstaller); this delivers the runnable
  desktop entry, not a distributable binary.
- Auto-serving the token to the desktop window (login screen handles auth).

---

# 5. Acceptance Criteria

- With a frontend build, `/` and client routes serve index.html; API 404s remain
  JSON.
- `desktop.py` opens a window when pywebview is installed, else runs the server
  with guidance.

---

# 6. Verification

- `test_frontend_serving.py`: root serves SPA; client route falls back; unknown
  API path is JSON 404. 100 backend tests pass. Ruff clean.

---

# 7. Next Milestone

End-to-end acceptance test + release readiness (M13.2).

---

# 8. Completion Record

**Date:** 2026-07-08

The same Flask app serves both the API and the built SPA, so the desktop
container is just a window over localhost. The catch-all SPA route explicitly
excludes API paths to preserve the JSON error contract (ADR-012).
