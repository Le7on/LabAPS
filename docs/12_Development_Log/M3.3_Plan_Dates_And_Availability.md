# Development Log

## Milestone M3.3 — Plan Dates & Per-Plan Availability

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-11

---

# 1. Objective

Let a Plan carry a real start/end date range and shift mode (via date pickers),
and let each Plan mark specific staff/equipment available or unavailable for its
period.

---

# 2. Scope

- Plan create form uses **start/end date pickers** + **shift mode**; the list
  shows the date range. (Backend already modelled start_date/end_date/shift_mode;
  the form now sends them.)
- Per-plan availability: `plan_resource_availability` table + repository + API
  (`GET/POST /plans/{id}/availability`); default available, stores exceptions.
- Scheduling console gains an **Availability** step: toggle each staff/equipment
  for the selected plan before generating.
- `generate-instances` excludes resources marked unavailable for the plan.
- Solver fix: an empty resource set with a required resource is now correctly
  infeasible (previously silently scheduled).
