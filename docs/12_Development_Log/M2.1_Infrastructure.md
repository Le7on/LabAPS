# Development Log

## Milestone M2.1 — Infrastructure

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-07

---

# 1. Objective

Build the technical foundation for persistence: SQLAlchemy ORM base, database
engine and session management, the Unit of Work transaction boundary, and
Alembic migrations.

Implements Phase 2 of the Bootstrap Plan and the SQLAlchemy Mapping Guide.

---

# 2. Scope

- ORM declarative base + `BaseEntity` (UUID primary key, audit and timestamp
  columns), per the SQLAlchemy Mapping Guide (UUID identity, audit fields).
- Database engine + session factory (`Database`), SQLite-aware.
- Unit of Work context manager owning the transaction boundary
  ("One Use Case = One Unit of Work", DI Strategy section 10).
- Alembic wired to `Base.metadata` with an environment-aware URL.

---

# 3. Deliverables

| Component        | Location                                             |
| ---------------- | ---------------------------------------------------- |
| ORM base         | `backend/infrastructure/orm/common/base.py`          |
| Database/session | `backend/infrastructure/persistence/database.py`     |
| Unit of Work     | `backend/infrastructure/persistence/unit_of_work.py` |
| Alembic config   | `alembic.ini`, `migrations/env.py`                   |
| Migrations       | `migrations/versions/*`                              |

---

# 4. Out of Scope

- Concrete aggregates and their tables (delivered per module: M3.1, M5.1).
- Query Service / read models (later).

---

# 5. Acceptance Criteria

- Database migration executes successfully (`alembic upgrade head`).
- ORM models register on `Base.metadata`.
- A Unit of Work commits on success and rolls back on error.

---

# 6. Verification

- `alembic revision --autogenerate` detected tables; `alembic upgrade head`
  applied them (verified for plan, plan_version and equipment).
- Unit of Work exercised by all use-case tests (M3.1, M5.1).
- Ruff clean; `migrations/` excluded from lint (tool-generated).

# 8. Completion Record

**Date:** 2026-07-07

The Unit of Work was introduced to remove repeated session/commit/rollback/close
boilerplate from use cases; it exposes per-aggregate repositories. ORM models
live in `infrastructure/orm/<module>/` and are converted to/from domain objects
by repositories, keeping the domain framework-free.
