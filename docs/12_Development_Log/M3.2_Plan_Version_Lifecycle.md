# 1. Objective

Implement the Plan Version lifecycle (Working -> Scheduled -> Reviewed ->
Published -> Archived) as domain behaviour, and enforce published immutability
and the single-published-version invariant in code rather than by convention.

Implements the State Model (chapter 3) and its business rules BR-PV-001..006.

---

# 2. Scope

- `PlanVersionStatus` aligned to the documented states (Working, Scheduled,
  Reviewed, Published, Archived).
- `PlanVersion` behaviour: `mark_scheduled`, `mark_reviewed`, `publish`,
  `archive`, each validating the current state and raising `ConflictError` with
  the documented codes (`INVALID_PLAN_VERSION_STATE`, `PLAN_VERSION_PUBLISHED`).
- `Plan` aggregate coordinates transitions and enforces BR-PV-004 (publishing a
  version archives any previously published one).
- Generate Schedule marks the version Scheduled on a feasible result (BR-PV-002)
  and rejects scheduling a Published version.
- Repository `save` now syncs existing versions' mutable state (status), not just
  new versions.
- Use cases: Review / Publish / Archive Plan Version.
- API: `POST /plans/{id}/versions/{vid}/{review|publish|archive}`.

# 4. Out of Scope

- Domain events (ScheduleGenerated, PlanVersionPublished, ...) — documented but
  deferred until an event bus is introduced.
- Making Assignments executable on publish (Execution module, later).

---

# 5. Acceptance Criteria

- Transitions follow Working -> Scheduled -> Reviewed -> Published; invalid
  transitions raise conflicts (HTTP 409).
- A Published version cannot be re-scheduled (immutability enforced in code).
- Only one Published version exists per Plan; publishing supersedes the previous.

# 8. Completion Record

**Date:** 2026-07-08

This addresses a design gap flagged earlier: published immutability was only a
documented convention. It is now enforced by the domain (behaviour-only
transitions, no public status setter) and surfaced through the API with the
documented business error codes. The repository was fixed to persist status
changes on existing versions, which the previous append-only `save` ignored.
