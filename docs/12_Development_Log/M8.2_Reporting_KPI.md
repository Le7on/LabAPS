# Development Log

## Milestone M8.2 — Reporting KPI & Equipment Utilization

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-08

---

# 1. Objective

Add read-only KPI reporting: an assignment status breakdown and a per-equipment
utilization proxy (assignment count and total planned busy time).

---

# 2. Scope

- `KpiQueryService`: assignment status breakdown (group-by status) and equipment
  utilization (outer join equipment to assignments, count + summed duration).
- `GetKpiUseCase` (read-only).
- API: `GET /api/v1/reports/kpi`.

---

# 3. Deliverables

| Component     | Location                                           |
| ------------- | -------------------------------------------------- |
| Query service | `backend/modules/reporting/query/kpi_query.py`     |
| Use case      | `backend/modules/reporting/application/get_kpi.py` |
| API           | `backend/modules/reporting/api/dashboard_api.py`   |
| Tests         | `backend/modules/reporting/tests/test_kpi_api.py`  |

---

# 4. Out of Scope

- True utilization against calendar capacity (busy time is a proxy until a
  calendar model exists).
- Material forecast and export reports.

---

# 5. Acceptance Criteria

- Status breakdown returns counts per assignment status.
- Equipment utilization returns count and busy time per equipment, including
  equipment with no assignments.
- Reporting performs no writes.

---

# 6. Verification

- `test_kpi_api.py`: empty KPI; after scheduling, status has one pending
  assignment and the equipment shows count 1 / busy time 3.
- 69 tests pass. Ruff clean. No schema change (reads existing tables).

---

# 7. Next Milestone

Calendar-based capacity for real utilization; material forecast; a KPI frontend
panel.

---

# 8. Completion Record

**Date:** 2026-07-08

Follows the read-only Query Service pattern established in M8.1. Utilization is a
