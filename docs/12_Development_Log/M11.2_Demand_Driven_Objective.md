# Development Log

## Milestone M11.2 — Demand-Driven Objective

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-08

---

# 1. Objective

Replace the interim pure-makespan objective with a demand-driven one: when a Plan
Version has demand, minimize demand-weighted completion time so higher-priority
demand is scheduled earlier. Realizes the Objective Model split (ADR-007) now
that Demand exists (M11.1).

---

# 2. Scope

- Scheduling model gains an `Objective` enum (MAKESPAN, WEIGHTED_COMPLETION) and
  a per-`Task` `weight`; `SchedulingModel.objective` selects the goal.
- OR-Tools adapter minimizes `sum(weight * end)` for WEIGHTED_COMPLETION, else
  makespan; makespan is always computed and reported.
- PlanningProblem `Operation` gains `weight`; `PlanningPolicies` gains
  `objective`; the model builder threads both through.
- Schedule-from-workflow: if the version has demand, weight operations by
  `sum(priority.weight * quantity)` and use WEIGHTED_COMPLETION; otherwise
  makespan.

# 4. Out of Scope

- Per-operation / per-project demand attribution (aggregate demand weight applies
  uniformly to the workflow's operations until Operation Instances tie to
  projects).
- Multi-objective weighting (utilization + balance) and a Solver Profile.

---

# 5. Acceptance Criteria

- MAKESPAN behaviour is unchanged (weights ignored).
- Existing scheduling tests still pass.
