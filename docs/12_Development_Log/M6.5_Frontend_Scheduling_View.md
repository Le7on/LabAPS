# Development Log

## Milestone M6.5 — Frontend Scheduling View

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-08

---

# 1. Objective

Add a Scheduling view that ties the whole pipeline together in the UI: pick a
plan and a workflow, create a version and schedule it, then review/publish and
drive assignment execution.

---

# 2. Scope

- `src/api/plans.js`: create version, schedule-from-workflow, list assignments,
  review, publish.
- `src/api/executions.js`: start / complete / fail / cancel.
- `src/views/SchedulingView.vue`: plan + workflow selectors, schedule action,
  review/publish action, assignments table with per-row start/complete actions.
- Route `/scheduling`; nav link in the app shell.

---

# 3. Deliverables

| Component       | Location                                  |
| --------------- | ----------------------------------------- |
| Plans API       | `frontend/src/api/plans.js`               |
| Executions API  | `frontend/src/api/executions.js`          |
| Scheduling view | `frontend/src/views/SchedulingView.vue`   |
| Router / shell  | `frontend/src/router/index.js`, `App.vue` |

---

# 4. Out of Scope

- Gantt/timeline visualization; fail/cancel UI (start/complete only).
- Selecting an existing version (a new version is created per schedule run).

---

# 5. Acceptance Criteria

- Scheduling creates a version, runs schedule-from-workflow and lists
  assignments.
- Review + publish moves assignments to Ready; per-row Start/Complete drive
  execution.
- The frontend builds and passes ESLint + Prettier.

---

# 6. Verification

- `vite build` succeeds (100 modules); ESLint + Prettier clean.

---

# 7. Next Milestone

Gantt/timeline visualization and fail/cancel controls; select an existing
version instead of always creating one.

---

# 8. Completion Record

**Date:** 2026-07-08

This closes the end-to-end UI loop across the backend milestones: Laboratory
(equipment/staff/workflows) -> Planning (plan/version) -> Scheduling (OR-Tools)
-> Execution (assignment lifecycle), all reachable from the SPA through the
envelope-based REST API.
