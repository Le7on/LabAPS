# Development Log

## Milestone M4.9 — FV (Equipment Validation) Validity Constraint

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-11

---

# 1. Objective

Enforce that every machine is validated periodically and that normal operations
run only while a machine's FV is in validity (ADR-019).

---

# 2. Scope

- Equipment gains `fv_duration` (default 1) and `fv_validity` (default 14;
  0 = exempt), across entity/ORM/repository/DTO/use cases and the equipment form.
- Scheduling: `SchedulingResource.fv_intervals` tiles the horizon with fixed FV
  occupancy per equipment; the solver adds these to each machine's no-overlap
  set, so normal work is pushed into valid windows. FV validity binding falls out
  of the existing resource constraint — no bespoke solver logic.
- Snapshot + schedule-instances thread FV params through.
