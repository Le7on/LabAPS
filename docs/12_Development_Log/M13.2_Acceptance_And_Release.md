# 1. Objective

Prove the whole product works together with a single end-to-end acceptance test,
and capture release readiness.

---

# 2. Scope

- `backend/tests/test_acceptance_e2e.py`: authenticated full-pipeline test —
  project + equipment + skilled/qualified staff, workflow definition (capability
  - skill + qualification), plan/version, demand, generate instances + snapshot,
    schedule, review/publish, execute (start -> complete) with audit history, and
    reporting (dashboard + KPI) reflecting the run.
- `docs/RELEASE_READINESS.md`: capability completeness, quality gates, run
  instructions, known limitations, go/no-go.
