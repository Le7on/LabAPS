# Development Log

## Milestone M6.6 — Frontend Overhaul & New Capability Coverage

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-08

---

# 1. Objective

Bring the SPA up to date with the backend: a clean design system + sidebar
layout, a new Projects view, availability/qualification entry on Equipment/Staff,
required-qualification on workflow operations, and an instance-based scheduling
workspace (replacing the removed schedule-from-workflow flow).

---

# 2. Scope

- Design system: `src/styles/theme.css` (tokens + card/button/table/form/badge
  classes); sidebar layout in `App.vue`.
- Dashboard redesigned as metric cards.
- Projects view (+ laboratory api/store additions).
- Equipment: availability windows entry; Staff: skills, qualifications
  (name:expiry) and availability. Shared parsing helpers in `utils/parse.js`.
- Workflow Definitions: dynamic operation rows incl. required qualification.
- Scheduling workspace: create version + generate instances, add demand
  (project/quantity/priority), set frozen window, run the scheduler, view
  assignments (equipment + staff) and drive execution (start/complete/fail/
  cancel), review & publish.
- Plans API client updated to instance-based endpoints (generate-instances,
  schedule-instances, demands).

---

# 3. Deliverables

| Component       | Location                                                                       |
| --------------- | ------------------------------------------------------------------------------ |
| Design system   | `frontend/src/styles/theme.css`                                                |
| Layout          | `frontend/src/App.vue`                                                         |
| Parsing helpers | `frontend/src/utils/parse.js`                                                  |
| Views           | dashboard, projects, equipment, staff, workflow-definitions, plans, scheduling |
| API/stores      | `api/plans.js`, `api/laboratory.js`, `stores/laboratory.js`                    |

---

# 4. Out of Scope

- A true Gantt/timeline chart (assignments shown as a table with start/end).
- Auth login UI (auth runs with tokens; the SPA is used with auth disabled in
  dev, or a token injected at the client — a login screen is a later milestone).
- Editing/deleting existing records.

---

# 5. Acceptance Criteria

- Every current backend capability is reachable from the UI: projects, demand,
  equipment/staff with availability + qualifications, workflow definitions with
  required qualification, instance generation, scheduling with a frozen window,
  and execution actions.
- The frontend builds and passes ESLint + Prettier.

---

# 6. Verification

- `vite build` succeeds (102 modules); ESLint + Prettier clean.
- Backend suite unaffected (93 tests pass).

---

# 7. Next Milestone

Gantt/timeline visualization; a login screen once auth is enabled in the desktop
shell; edit/deactivate actions.

---

# 8. Completion Record

**Date:** 2026-07-08

The UI is now aligned with the backend feature set behind a dependency-free
design system (no component library, so no new install needed while the network
was unavailable). The scheduling workspace walks the full pipeline end to end:
version -> instances -> demand -> schedule (with frozen window) -> assignments ->
execution.
