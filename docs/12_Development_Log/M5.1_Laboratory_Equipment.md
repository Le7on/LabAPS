# Development Log

## Milestone M5.1 — Laboratory Equipment Slice

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-07

---

# 1. Objective

Deliver the first slice of the Laboratory Definition domain: Equipment, from
domain entity through repository, use cases and REST API.

Implements ADR-010 (Laboratory Definition and Planning are separate domains).

---

# 2. Scope

- Domain entity: `Equipment` (equipment code, name, capability set, active flag).
- ORM model + repository (ORM <-> domain conversion).
- Use cases: create equipment, list equipment.
- REST API: `POST/GET /api/v1/equipment`.
- Alembic migration for the equipment table.

---

---

# 4. Out of Scope

- Staff, Workflow Definition, Project, Material, Capability, Skill.
- A normalized equipment_capability mapping table (capabilities are stored as a
  JSON list for this slice; the mapping table follows the Capability aggregate).

---

# 5. Acceptance Criteria

- Equipment can be created and listed through the API.
- Missing equipment code is rejected with a validation error.

---

# 6. Verification

- `test_equipment_api.py`: create + list; code validation (422).
- Alembic migration chain verified (plan -> equipment).
