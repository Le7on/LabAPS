# Development Log

## Milestone M10.2 — Frontend Authentication

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-08

---

# 1. Objective

Let the SPA authenticate against the token-based backend (ADR-013): a login
screen, a persisted token, and the current user shown in the shell.

---

# 2. Scope

- Axios client attaches the stored bearer token to every request; token stored
  in localStorage (`setToken`/`getToken`).
- `stores/auth.js`: restore on startup (via whoami), login (validate token),
  logout.
- `views/LoginView.vue`: token entry; public route `/login`.
- App shell shows current user + role and a sign-out button, or a sign-in link.

---

# 3. Deliverables

| Component    | Location                                  |
| ------------ | ----------------------------------------- |
| Client token | `frontend/src/api/client.js`              |
| Auth API     | `frontend/src/api/auth.js`                |
| Auth store   | `frontend/src/stores/auth.js`             |
| Login view   | `frontend/src/views/LoginView.vue`        |
| Shell        | `frontend/src/App.vue`, `router/index.js` |

---

# 4. Out of Scope

- Username/password login (backend issues opaque tokens; ADR-013).
- Route guards that hard-block unauthenticated access (backend enforces auth;
  when AUTH_ENABLED is off, the app is usable without a token).

---

# 5. Acceptance Criteria

- A valid token signs in and shows the user; an invalid token is rejected.
- The token persists across reloads and is attached to API calls.
- Sign-out clears the token.

---

# 6. Verification

- vite build succeeds (108 modules); ESLint + Prettier clean.

# 8. Completion Record

**Date:** 2026-07-08

The client attaches the token globally, so all existing API modules gain auth
without change. Login validates via whoami and stores the token; the shell
reflects the session. This pairs with the backend AUTH_ENABLED flag: enabled in
production, off for the dev/test convenience path.
