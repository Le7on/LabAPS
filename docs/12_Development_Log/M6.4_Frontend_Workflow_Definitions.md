# Development Log

## Milestone M6.4 — Frontend Workflow Definitions View

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-08

---

# 1. Objective

Add a Workflow Definitions view that lists workflows and creates one with a
dynamic list of operation definitions (type, duration, required capability,
required skill), consuming the laboratory API.

---

# 2. Scope

- `src/api/laboratory.js`: list/create workflow definitions.
- `src/stores/laboratory.js`: workflows state with fetch/add actions.
- `src/views/WorkflowDefinitionsView.vue`: create form with add/remove operation
  rows + list table.
- Route `/workflow-definitions`; nav link in the app shell.

---

# 3. Deliverables

| Component      | Location                                         |
| -------------- | ------------------------------------------------ |
| Laboratory API | `frontend/src/api/laboratory.js`                 |
| Store          | `frontend/src/stores/laboratory.js`              |
| Workflows view | `frontend/src/views/WorkflowDefinitionsView.vue` |
| Router / shell | `frontend/src/router/index.js`, `App.vue`        |

---

# 4. Out of Scope

- Editing/removing existing workflows or operations.
- A scheduling view that triggers schedule-from-workflow and renders assignments
  (next frontend milestone).

---

# 5. Acceptance Criteria

- Workflows can be created with multiple operations and listed.
- Empty operation rows are dropped; optional capability/skill omitted when blank.
- The frontend builds and passes ESLint + Prettier.

---

# 6. Verification

- `vite build` succeeds (97 modules); ESLint + Prettier clean.

---

# 7. Next Milestone

A scheduling view: pick a plan version + workflow, run schedule-from-workflow,
render assignments (equipment + staff) and drive execution actions.

---

# 8. Completion Record

**Date:** 2026-07-08

Reuses the single laboratory store and the envelope-unwrapping Axios client. The
operation-row editor keeps the create payload aligned with the workflow
definition API (operationType/duration/requiredCapability/requiredSkill).
