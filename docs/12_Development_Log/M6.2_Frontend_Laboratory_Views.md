# Development Log

## Milestone M6.2 — Frontend Laboratory Views

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-08

---

# 1. Objective

Extend the Vue 3 SPA with Equipment and Staff management views that consume the
Laboratory REST API, so the backend's laboratory features are usable from the UI.

---

# 2. Scope

- `src/api/laboratory.js`: list/create for equipment and staff.
- `src/stores/laboratory.js`: Pinia store holding equipment and staff with
  fetch/create actions and error state.
- `src/views/EquipmentView.vue`, `src/views/StaffView.vue`: list tables + create
  forms (comma-separated capabilities/skills parsed into arrays).
- Routes `/equipment` and `/staff`; navigation links in the app shell.

---

# 3. Deliverables

| Component      | Location                                  |
| -------------- | ----------------------------------------- |
| Laboratory API | `frontend/src/api/laboratory.js`          |
| Store          | `frontend/src/stores/laboratory.js`       |
| Equipment view | `frontend/src/views/EquipmentView.vue`    |
| Staff view     | `frontend/src/views/StaffView.vue`        |
| Router / shell | `frontend/src/router/index.js`, `App.vue` |

---

# 4. Out of Scope

- Workflow Definition editor and a scheduling/Gantt view (next frontend
  milestone).
- Edit/deactivate actions (create + list only for this slice).

---

# 5. Acceptance Criteria

- Equipment and Staff can be created and listed from the UI.
- Capabilities/skills entered as comma-separated text become arrays in the API
  call.
- The frontend builds and passes ESLint + Prettier.

---

# 6. Verification

- `vite build` succeeds (92 modules).
- ESLint + Prettier clean.
- Views reuse the envelope-unwrapping Axios client (ADR-012), so they consume
  `data`/`meta` directly.

---

# 7. Next Milestone

Workflow Definition editor and a scheduling view that triggers
schedule-from-workflow and renders assignments (equipment + staff) on a timeline.

---

# 8. Completion Record

**Date:** 2026-07-08

The views follow the PlansView pattern (api module + Pinia store + view + route).
A single laboratory store serves both entities to keep the shape consistent as
more laboratory resources are added.
