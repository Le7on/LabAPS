# Development Log

## Milestone M11.4 — Relational Domain Model in the Frontend

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-11

---

# 1. Objective

Make the UI use the relational domain the backend already supported, replacing
free-text fields with dropdown/multi-select relations, and add per-Method run
counts.

---

# 2. Scope

- Workflows: choose Project (dropdown); add Methods with work-hours, equipment
  multi-select, and prerequisite (depends-on) multi-select. Method dependencies
  are expressed by method name and resolved to instance ids at generation.
- Staff: multi-select **qualified projects** (= skills, many-to-many).
- Equipment: multi-select **methods it can run** (many-to-many).
- Scheduling: per-Method **runs per method** counts before generate; backend
  `generate-instances` accepts `runCounts` and materializes each method N times,
  preserving the dependency DAG (run r depends on run r of prerequisites).

---
