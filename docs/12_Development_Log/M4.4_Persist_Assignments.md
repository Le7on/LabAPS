# Development Log

## Milestone M4.4 — Persist Assignments

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-08

---

# 1. Objective

Persist scheduling results (Assignments) onto the Plan Version instead of only
returning them, and expose them for retrieval.

Assignments belong to the Plan Version aggregate (Table Dictionary: assignment).

---

# 2. Scope

- `AssignmentORM`: assignment table (plan_version_id, operation_id, equipment_id,
  staff_id, planned_start, planned_end, status).
- `AssignmentRepository`: replace-for-version (delete-then-insert on each run)
  and list-for-version.
- Unit of Work exposes `uow.assignments`.
- Schedule-from-workflow persists assignments on a feasible run.
- API: `GET /plans/{id}/versions/{vid}/assignments`.
- Alembic migration for the assignment table.

---

# 3. Deliverables

| Component       | Location                                                                  |
| --------------- | ------------------------------------------------------------------------- |
| ORM             | `backend/infrastructure/orm/planning/assignment_orm.py`                   |
| Repository      | `backend/modules/planning/repository/assignment_repository.py`            |
| Use case        | `backend/modules/planning/application/list_assignments.py`                |
| Schedule wiring | `backend/modules/planning/application/generate_schedule_from_workflow.py` |

| Tests           | `backend/modules/planning/tests/test_schedule_from_workflow_api.py`       |

---

# 4. Out of Scope

- Operation Instances (assignment references the scheduling operation id for now).
- Datetime scheduling / calendars (planned start/end are integer horizon units).
- Assignment execution status transitions (Execution module).

---

# 5. Acceptance Criteria

- A feasible schedule persists one assignment per scheduled operation on the
  version.
- Re-running scheduling replaces the version's prior assignments.
- Assignments are retrievable via the API.

---

# 6. Verification

- `test_schedule_from_workflow_api.py`: assignments persisted and retrievable
  (count, status "planned", timings).
- Alembic autogenerate detected the assignment table; upgrade head applied it.
- 56 tests pass. Ruff clean.

# 8. Completion Record

**Date:** 2026-07-08

Closes the loop flagged in M4.2/M4.3: scheduling results are now durable.
Assignments are owned by the Plan Version and replaced wholesale on each
scheduling run, which matches "re-running the solver regenerates the working
version" (State Model). Times are stored as integer horizon units until a
calendar model lands.
