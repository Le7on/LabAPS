# Development Log

## Milestone M11.1 — Demand Domain

**Project:** Laboratory Advanced Planning & Scheduling Platform (Lab APS)

**Version:** 1.0

**Status:** Complete

**Owner:** Development Team

**Completed:** 2026-07-08

---

# 1. Objective

Model Demand (requested production quantities) as part of the Plan Version
aggregate, so scheduling can optimize toward business demand (Table Dictionary:
demand). Prerequisite for the demand-driven objective (M11.2).

---

# 2. Scope

- `Demand` entity + `DemandPriority` enum (low/normal/high with weights 1/2/4).
- `DemandORM` (plan_version_id, project_id, quantity, priority), owned by the
  Plan Version.
- `DemandRepository` (add, list for version); wired into the Unit of Work.
- Use cases: add demand, list demand for a version.
- API: `POST/GET /api/v1/plans/{id}/versions/{vid}/demands`.
- Alembic migration for the demand table.

---

# 4. Out of Scope

- Project aggregate (project_id is a free reference for now).
- Demand fulfilment tracking against produced quantities.

---

# 5. Acceptance Criteria

- Demand can be added to a plan version and listed.
- Quantity must be positive; unknown priority rejected (422).
- Adding demand to an unknown plan/version returns 404.

---
